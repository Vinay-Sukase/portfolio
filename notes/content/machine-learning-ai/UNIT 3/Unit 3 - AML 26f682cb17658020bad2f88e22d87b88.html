Clustering

Clustering is a unsupervised learning technique designed to group unlabeled examples based on their similarity or dissimilarity to each other.

It is widely used in market segmentation, image segmentation, anomaly detection, and pattern recognition

Key Concepts

  • Cluster: A collection of data points with high similarity.
  • Centroid: A representative point (like the mean) of a cluster.
  • Distance Metrics: Common measures of similarity/dissimilarity include:
    • Euclidean Distance : d=(x2−x1)2+(y2−y1)2d= \sqrt{(x_2-x_1)^2+(y_2-y_1)^2}d=(x2​−x1​)2+(y2​−y1​)2​﻿
    • Manhattan Distance : ∣(x2−x1)∣+∣(y2−y1)∣|(x_2-x_1)|+|(y_2-y_1)|∣(x2​−x1​)∣+∣(y2​−y1​)∣﻿
    • Cosine Similarity

Advantages

  1. Helps in data summarization and pattern discovery
  1. Does not require labeled data
  1. Can handle large datasets with proper algorithms

Disadvantages

  1. Requires choosing number of clusters (k) in some algorithms
  1. Sensitive to outliers and noise
  1. Results may vary depending on distance metrics or algorithm initialization

AGNES

AGNES is a Hierarchical Clustering algorithm that uses a bottom-up approach. It starts with each data point as a single cluster and successively merges the closest clusters until all data points belong to one cluster or a stopping criterion is met.

  • Agglomerative: Means “coming together” – clusters are merged iteratively.
  • Distance Metrics: Determines which clusters to merge. Common metrics include:
    • Single linkage (minimum distance)
    • Complete linkage (maximum distance)
    • Average linkage (average distance)
    • Ward’s method (minimizes total variance)
  • Dendrogram: A tree-like diagram representing the merging process, showing the hierarchy of clusters.

Algorithm Steps (Bottom-Up)

  1. Start: Treat each data point as a separate cluster.
  1. Compute Distances: Calculate the pairwise distance between all clusters.
  1. Merge Clusters: Merge the two clusters with the smallest distance.
  1. Update Distance Matrix: Recalculate distances between the new cluster and existing clusters using a chosen linkage method.
  1. Repeat Steps 3-4: Continue until all points are merged into a single cluster or the desired number of clusters is obtained.
  1. Create Dendrogram: Visualize the hierarchy of clusters.

Advantages

  1. Produces a hierarchical tree (dendrogram) showing cluster relationships.
  1. No need to specify the number of clusters in advance.
  1. Works well for small to medium datasets.

Disadvantages

  1. Computationally expensive for large datasets (O(n3)O(n^3)O(n3) time complexity).
  1. Sensitive to outliers.
  1. Once a merge is done, it cannot be undone (no backtracking).

DIANA

DIANA (Divisive Analysis) is a Hierarchical Clustering algorithm that follows a top-down approach.

It starts with all data points in one big cluster and repeatedly splits clusters into smaller clusters until each object becomes its own cluster or a stopping condition is met.

Key Concepts

  • Divisive Clustering: Start with one cluster → split into many.
  • Dissimilarity: Distance or difference between points, used to decide splits.
  • Splinter Group: A subgroup that “splinters” away from the main cluster based on large dissimilarity.
  • Dendrogram: A tree diagram showing how the cluster splits occur at each level.

Steps of the DIANA Algorithm (Top–Down)

  1. Start with all data points in one cluster.
  1. Compute dissimilarity (distance) between all points.
  1. Find the point most dissimilar from the others → this becomes the first member of the splinter group.
  1. Add other points to the splinter group if they are closer to the splinter group than to the main group.
  1. Continue splitting clusters until:
    • Every point is separate, OR
    • Desired number of clusters is reached.

Advantages

  1. Produces clear hierarchical structure (dendrogram).
  1. Good when natural clusters are well-separated.
  1. Opposite structure to AGNES → helps compare bottom-up vs top-down clustering.

Disadvantages

  1. Computationally expensive for large datasets.
  1. More complex splitting decisions than merging (AGNES).
  1. Sensitive to noise/outliers.

K-Means Clustering

K-Means is a popular unsupervised learning algorithm used to partition data into k clusters, where each data point belongs to the cluster with the nearest mean (centroid).

The goal is to minimize the sum of squared distances between points and their cluster centroids.

Key Concepts

  • k: Number of clusters to form (chosen by the user).
  • Centroid: The mean position of all points in a cluster.
  • Distance Metric: Usually Euclidean distance.
  • Objective: Minimize the within-cluster sum of squares (WCSS):

K-Means Algorithm Steps

  1. Choose number of clusters (k)

    Example: k = 2

  1. Initialize centroids
    • Randomly select k points as initial centroids
  1. Assignment Step
    • Assign each data point to the closest centroid
  1. Update Step
    • Recalculate centroids as the mean of all points in each cluster
  1. Repeat Steps 3–4
    • Until centroids do not change significantly (algorithm converges)

Advantages

  1. Simple and easy to implement
  1. Fast and efficient for large datasets
  1. Works well when clusters are spherical and well separated

Disadvantages

  1. Must specify k in advance
  1. Sensitive to initial centroid positions
  1. Struggles with non-spherical clusters or clusters of different sizes
  1. Sensitive to outliers

K-Modes Clustering

K-Modes is an unsupervised learning clustering algorithm used to partition categorical (non-numeric) data into k clusters, where each data point belongs to the cluster with the nearest mode.

It is a categorical-data counterpart of K-Means.

Instead of using mean and Euclidean distance, K-Modes uses mode and simple matching dissimilarity.

The goal is to minimize the total dissimilarity between data points and their cluster modes.

Key Concepts

  • k: Number of clusters to form (chosen by the user).
  • Mode: Most frequent category value in a cluster (cluster center).
  • Distance Metric: Simple Matching Distance.
  • Objective: Minimize within-cluster dissimilarity:

Where:

  • CiC_iCi​﻿ = ithi^{th}ith﻿ cluster
  • mim_imi​﻿ = mode of the ithi^{th}ith﻿ cluster
  • d(x,mi)d(x,m_i)d(x,mi​)﻿ = number of mismatched attributes

K-Modes Algorithm Steps

  1. Choose number of clusters (k)

    Example: k = 2

  1. Initialize modes
    • Randomly select k data points as initial modes
  1. Assignment Step
    • Assign each data point to the cluster whose mode has minimum dissimilarity
  1. Update Step
    • Recalculate the mode of each attribute in every cluster
  1. Repeat Steps 3–4
    • Until modes do not change (algorithm converges)

Advantages

  1. Suitable for categorical data
  1. Simple and fast
  1. No need to convert categories into numbers
  1. Scales well for large datasets

Disadvantages

  1. Must specify k in advance
  1. Sensitive to initial modes
  1. Cannot handle numerical data directly
  1. May converge to local optima

Naïve Bayes Classifier

Naïve Bayes is a probabilistic classification algorithm based on Bayes’ Theorem with the assumption that features are conditionally independent given the class.

It is widely used in text classification, spam detection, sentiment analysis, and medical diagnosis.

Types of Naïve Bayes Classifiers

  1. Multinomial Naïve Bayes → used for text classification
  1. Bernoulli Naïve Bayes → binary features (word present/absent)
  1. Gaussian Naïve Bayes → continuous features assumed to follow a normal distribution

Advantages

  1. Very fast training and prediction
  1. Works well with high-dimensional data (e.g., text)
  1. Requires small amount of training data
  1. Performs well even when independence assumption is not fully true

Disadvantages

  1. Assumes feature independence (rarely true in real data)
  1. Zero-frequency problem (solved by Laplace smoothing)
  1. Not suitable for datasets where relationships between features are important
